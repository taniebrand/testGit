About Decentralised Data Storage:

Building the decentralized web with privacy in mind

What would you help build:
Permissionless P2P protocol that allows incentivised data storage and a storage marketplace. Two main use-cases:

  • 1 - Decentralised data storage and CDN

    allows unstoppable decentralised apps or decentralised scalable content distribution (e.g. Netflix, Spotify…). Users pay to consume content.

  • 2 - Personal cloud storage

    - allows users to store their data encrypted in the P2P network. Users can choose how many replicas and where are they stored.

Anyone can participate in the network as a provider - listing their storage on the marketplace. The system will integrate into enterprise ready framework providing P2P payment, address resolver, P2P messaging and data oracles. You can read more in RIF Storage: A Decentralised Storage Solution and RIF Storage The First Chunks articles

Essential Duties:

As part of our Decentralised Data Storage Team, on a regular work day you will have the following responsibilities:

  • Together with IOV labs & Swarm build SWAP incentives system for the Swarm system (board, mvp specs)
  • Help integrate RIF Payments and other RIF services into Swarm (integration outlined in SWIP drafts, to be accepted)
  • Provide guidance to junior go devs and over time take technical ownership over the go development within RIF Storage
  • Help evaluate and implement marketplace integration with other storage systems (IPFS, SIA…)
  • Help define and work as needed on RIF Communications solution (utilises Swarm PSS & feeds protocols)
